## Affected Pages/Routes
- CLI Commands: `corivo init`, `corivo daemon start/stop/status`, `corivo update`, `corivo cold-scan`
- Daemon: macOS launchd integration
- Background: Heartbeat loop processing

## Key Interactions to Verify
- **Installation Flow**: `curl -fsSL https://corivo.ai | sh` → Cold Scan → First Push → Daemon start
- **Daemon Lifecycle**: `corivo daemon start` → launchd load → heartbeat running → `corivo daemon stop`
- **Update Flow**: Version check → download → checksum → atomic replace → rollback on failure
- **Cold Scan**: Extract user info from ~/.claude, ~/.gitconfig, package.json files

## Edge Cases
- **Daemon**: LaunchAgents directory doesn't exist, plist already loaded, daemon crashes during startup
- **Update**: Network timeout during download, checksum mismatch, insufficient disk space
- **Cold Scan**: File permissions denied, malformed JSON, file path doesn't exist
- **First Run**: Empty database after cold scan, no valid blocks found

## Critical Paths
1. **End-to-End Installation**: Fresh macOS arm64 environment → `curl | sh` → Corivo initialized with identity → daemon running → first heartbeat completes
2. **Update Recovery**: New version crashes → automatic rollback to previous version → daemon restarts with old binary
3. **Cold Scan Safety**: Sensitive files (credentials, tokens) are never scanned even if present in expected locations
